# Docs Guide

This file is the docs index.
Use it to decide which document set is active before opening individual docs.

## Folders

- `current/`
  - current working references only
  - docs that still directly drive implementation
  - see `current/README.md` for `Strongly Active / Context / Dormant But Kept`
- `specs/`
  - feature and system specs that may drive future implementation
- `reference/`
  - background notes, planning material, and lower-priority reference docs
  - see `reference/README.md` for moved asset/audio guidance
- `archive/`
  - completed fix notes
  - finished checklists
  - one-off migration/runbook memos kept only for history

## Docs Compression Rule

- Do not use this file as a giant flat reading list.
- `docs/current/active-doc-shortlist-2026-03-31.md` is the primary entry for day-to-day implementation work.
- `docs/current/README.md` decides which current docs are `Strongly Active`, `Context / Secondary`, or `Dormant But Kept`.
- Archive candidates should be removed from daily reading paths before they are physically moved to `docs/archive/`.
