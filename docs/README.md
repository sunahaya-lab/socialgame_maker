# Docs Guide

This file is the docs index.
Use it to decide which document set is active before opening individual docs.

## Folders

- `current/`
  - current working references only
  - docs that still directly drive implementation
- `specs/`
  - feature and system specs that may drive future implementation
- `reference/`
  - background notes, planning material, and lower-priority reference docs
- `archive/`
  - completed fix notes
  - finished checklists
  - one-off migration/runbook memos kept only for history
