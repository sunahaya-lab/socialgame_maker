# UIアセット サイズ規格ガイド

## 目的

このメモは、`socia_maker` に取り込む UI アセットを、外部ツールで制作する際の初期サイズ規格をまとめたものです。

主な用途:

- Affinity での UI 制作
- Figma での UI 制作
- `PNG` / `SVG` での書き出し
- `team_owned` 素材としての登録

## 基本方針

- 最初は `ボタン / バナー / 枠 / アイコン` を優先する
- サイズの種類を増やしすぎず、まずは共通規格で揃える
- 置いてすぐ使える完成パーツを先に作る
- 使い回す UI パーツは透過素材を前提にする

## ボタン

### 小ボタン

- サイズ: `160 x 56`
- 用途:
  - 小さな補助操作
  - 設定
  - 戻る

### 中ボタン

- サイズ: `220 x 72`
- 用途:
  - ホーム基本導線
  - ストーリー
  - コレクション

### 大ボタン

- サイズ: `320 x 88`
- 用途:
  - ガチャ
  - テスト戦闘
  - 主導線

### 横長 CTA ボタン

- サイズ: `420 x 96`
- 用途:
  - 10回引き
  - 編成確定
  - 強めの主ボタン

## バナー

## 最初に作るおすすめ順

まずは次の順で作るのがおすすめです。

1. `320 x 88` の大ボタン
2. `220 x 72` の中ボタン
3. `960 x 240` のメインバナー
4. `640 x 180` のサブバナー
5. `640 x 360` の汎用パネル
6. `300 x 72` のネームプレート
7. `64 x 64` の標準アイコン

## 制作時の注意

- 使い回すボタン、枠、装飾は透過 `PNG` を基本にする
- 単純なアイコンは `SVG` を優先する
- 重要な文字や絵を端ギリギリに置かない
- 内側の安全余白を揃える
- 実際に使うサイズから離れすぎた巨大書き出しは避ける
- 同カテゴリ内で角丸、影、ハイライトの強さを揃える
