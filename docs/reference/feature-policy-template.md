# Feature Policy Template

## 使い方

このテンプレは、機能ごとに以下を整理するためのもの。

- 無料版か有料版か
- 誰が使えるか
- 何ができるか
- 何を禁止するか
- UI でどう見せるか
- API でどう制御するか

このブロックを複数並べれば、そのまま機能カタログの土台として使える。

## 記入例 1

### 共同編集URL再発行

- 機能ID: `collab_share_rotate`
- カテゴリ: 共有
- 無料版 / 有料版: 無料版
- 優先度: 高

- 対象ユーザー: project owner
- 実行できる人: owner のみ
- 対象スコープ: project

- できること:
  - 新しい共同編集URLを発行できる
  - 以前の共同編集URLを失効できる
- できないこと:
  - 他人の project の URL は再発行できない
- 保存対象:
  - `project_collab_shares.active_token`
  - `project_collab_shares.version`
- 影響範囲:
  - 旧URLでのアクセスは無効になる

- UI表示:
  - 共有設定画面にボタン表示
- 無料版での見せ方:
  - 通常表示
- 有料版での見せ方:
  - 通常表示
- エラー時の見せ方:
  - 「再発行に失敗しました」を表示

- API:
  - `POST /api/collab-share/rotate`
- 成功条件:
  - owner が対象 project に対して実行する
- 拒否条件:
  - owner 以外
  - project 不存在
- 権限エラー時レスポンス:
  - `403`

- 補足:
  - 再発行前に確認ダイアログを出す
- 未決事項:
  - URL再発行履歴を保存するか

## 記入例 2

### 公開共有URL発行

- 機能ID: `public_share_create`
- カテゴリ: 共有
- 無料版 / 有料版: 有料版
- 優先度: 高

- 対象ユーザー: project owner
- 実行できる人: owner のみ
- 対象スコープ: project

- できること:
  - 編集不可の公開共有URLを発行できる
- できないこと:
  - 無料版 project では実行できない
  - 公開共有URLから編集はできない
- 保存対象:
  - `project_public_shares`
  - `project_license_states`
- 影響範囲:
  - 外部閲覧者が閲覧とプレイを行える

- UI表示:
  - 共有設定画面に表示
- 無料版での見せ方:
  - disabled 表示、またはアップグレード導線
- 有料版での見せ方:
  - 通常表示
- エラー時の見せ方:
  - 「この機能は有料版限定です」を表示

- API:
  - `POST /api/public-share/create`
- 成功条件:
  - owner であり、対象 project が有料版
- 拒否条件:
  - owner 以外
  - 無料版
- 権限エラー時レスポンス:
  - `403`

- 補足:
  - 初期実装では1 project につき1本でよい
- 未決事項:
  - ライブ参照にするか snapshot にするか

## カタログ化するときのおすすめ列

機能をたくさん並べて一覧化するときは、最低でも次の列があると管理しやすい。

- 機能ID
- 機能名
- カテゴリ
- 無料版 / 有料版
- 実行できる人
- 対象スコープ
- API
- 状態
- 備考

## これを機能カタログとして使うときの考え方

はい。これを複数並べれば、そのまま機能カタログになる。

ただし、より正確には次の 2 層に分けると運用しやすい。

1. 一覧表
- 全機能を俯瞰するためのカタログ

2. 個別詳細
- 各機能の詳細仕様

おすすめの構成はこう。

- `feature-catalog.md`
  - 一覧表
- `feature-policy-template.md`
  - 個別機能を書くテンプレ

必要なら次に、共有・課金・編集・公開の初期機能を並べた `docs/feature-catalog.md` のたたき台も作れる。
