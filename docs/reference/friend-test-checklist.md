# 友人テストチェックリスト

## 目的

- 少人数の友人に触ってもらい、「今ある機能で何が通じて何が迷われるか」を集める。
- バグ報告だけでなく、「ほしい機能」と「違和感」を洗い出す。

## テストの見方

- バグだけでなく、迷い・不安・分かりにくさも重要な発見として扱う。
- 「できなかったこと」より「どこで止まったか」を重視する。
- 作者の想定と、友人の自然な触り方のズレを観察する。

## 最初に見たい観点

### 1. 第一印象

- 何のアプリかすぐ分かるか
- 最初に何を押せばいいか分かるか
- 画面の情報量が多すぎないか

### 2. ホーム画面

- ホームがそれっぽく見えるか
- キャラ、会話、バナー、通貨などが自然に読めるか
- 「ホーム設定」「ホーム編集」「共有」の意味が伝わるか

### 3. ガチャ

- 引き方が直感的か
- 結果演出は最低限わかりやすいか
- 引いたカードがコレクションに反映されたと理解できるか

### 4. ストーリー

- どこから読めるか分かるか
- `NEW / LOCKED / CLEAR` 相当の状態が直感的か
- キャラストーリーの解放条件が伝わるか

### 5. コレクション

- 所持カード一覧として分かりやすいか
- 詳細を開きたい気持ちになるか
- 欲しい絞り込みや並び替えがあるか

### 6. 編集

- どこから編集に入るのか分かるか
- 保存できたか不安にならないか
- どのデータが project 共通で、どのデータが個人進行か混乱しないか

### 7. 共有

- 共有ボタンの意味が分かるか
- 共同編集URLと公開URLの違いが伝わるか
- 「何が共有されて何が共有されないか」が伝わるか

## 友人に聞きたい質問

- 最初に何をするアプリだと思ったか
- どこで迷ったか
- どこが楽しかったか
- どこが分かりにくかったか
- 先に欲しい機能は何か
- 人に見せたくなる要素はどこか

## 欲しい機能の拾い方

- 「あったら便利」より「無いと止まる」を優先する
- 似た要望はまとめる
- 要望を `無料版向け`, `有料版向け`, `全員向け改善` に分ける

## 記録フォーマット

### 1件ごとに残す項目

- テスター名
- 日付
- 触った範囲
- 止まった箇所
- 欲しい機能
- 印象に残った点
- 優先度

## 優先度の付け方

### 高

- 進行不能
- 保存が分からない
- 共有で誤解が起きる
- 何をしていいか分からず離脱する

### 中

- やれるが分かりにくい
- 言葉が伝わりにくい
- UI が怖い、重い、面倒

### 低

- 見た目の好み
- 将来あると嬉しい追加機能

## 友人テストのおすすめ順

1. ホームを見る
2. ガチャを引く
3. ストーリーを読む
4. コレクションを見る
5. 共有を触る
6. 編集を試す

## このチェックリストの使い方

- 最初は 3〜5 人の少人数で使う
- 毎回全部を聞かず、止まったポイントを中心に掘る
- 要望が増えたら `feature-catalog.md` に戻して整理する
