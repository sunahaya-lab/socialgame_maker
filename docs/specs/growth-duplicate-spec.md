# growth-duplicate-spec

## 目的
- 育成の拘束感を増やさず、`進化` 体験を主役にする
- 重複カードの扱いを自動処理にせず、ユーザー選択にする
- キャラカードと装備カードの両方で同じ思想を使う

## 適用範囲
- 対象:
  - キャラカード (`entries`)
  - 装備カード (`equipmentCards`)
- 非対象:
  - マスターデータの破壊的変更
  - 既存ガチャ保存データの強制移行

## 基本方針
1. 重複カードは自動変換しない
2. 使い道はユーザーが都度選ぶ
3. `凸` は失敗なし、確率なし
4. 育成数値は軽量に抑える
5. 進化演出・見た目変化の価値を上げる

## 用語
- `重複カード`:
  - 既に所持済みの同一カードを再入手した分
- `凸`:
  - 同一カードを素材にした限界突破
- `変換`:
  - 重複カードを任意資源へ交換する操作
- `進化`:
  - 段階的な見た目/演出の解放を伴う成長

## ユーザー行動仕様

### 重複入手時
- 重複カードはそのまま所持数に加算する
- 自動で `凸` や `ポイント変換` はしない

### 重複カードの使い道
- `凸に使う`
- `そのまま保持する`
- `変換する`

### 変換
- 変換対象と変換先はユーザーが選ぶ
- 変換先候補の初期案:
  - `共鳴ポイント`
  - `進化素材`
  - `ゴールド`
- 変換実行前にプレビューを表示:
  - 消費枚数
  - 獲得資源量
  - 実行後の残枚数

## 凸仕様
- 最大: `5凸`
- 消費: `1凸につき同一カード1枚`
- 成功率: `100%`
- 失敗・破損: `なし`
- 効果は軽量:
  - ステータスは小幅上昇
  - 見た目/演出/ボイス解放を重視

## 進化仕様
- 段階: `0 -> 1 -> 2 -> 3` (初期案)
- 進化素材は軽量設計:
  - `進化素材` と `ゴールド` のみ
- 進化で強化するもの:
  - イラスト枠演出
  - ホーム演出
  - 詳細画面の進化演出
- 数値の増分は控えめ

## 強化仕様 (簡易)
- レベル強化はワンタップ式
- 素材は最小種別で運用
- `必要素材が多すぎて何もできない` 状態を避ける

## 編成・詳細導線

### 画面名称
- 既存 `編成` は `キャラ・編成` に変更する

### 操作
- 短押し:
  - 編成の選択/入れ替え
- 長押し (`450ms` 初期値):
  - 詳細画面へ遷移

### 詳細画面
- キャラ詳細と装備詳細を分離
- 共通タブ案:
  - `概要`
  - `強化`
  - `進化`
  - `凸`
  - `変換`

## 重複保持と編成自由度
- 重複カードは保持可能
- 同一カードの複数編成は将来拡張として許可可能にする
- 初期実装では次のどちらかを切替可能にする想定:
  - `同一カードは1枠のみ`
  - `所持数の範囲で複数枠可`

## データ方針
- 共有マスター (`entries`, `equipmentCards`) は変更しない
- 進行データはプレイヤー状態へ追加する

初期データ案:
```js
playerState.cardGrowth[cardId] = {
  level: 1,
  evolveStage: 0,
  limitBreak: 0,
  lockedCopies: 0
}

playerState.equipmentGrowth[equipmentId] = {
  level: 1,
  evolveStage: 0,
  limitBreak: 0,
  lockedCopies: 0
}
```

補足:
- 所持数本体は既存 inventory 系を利用
- `lockedCopies` は誤変換防止用

## API方針 (新規)
- `POST /api/player-growth-action`
  - `enhance`
  - `evolve`
  - `limit_break`
  - `convert_duplicates`
  - `lock_copy`
  - `unlock_copy`
- すべて `project + user` スコープ

## UXガード
- 重要操作は確認ダイアログを表示
- `ロック済み` は凸素材/変換素材に出さない
- 操作結果は必ずトースト表示する

## バランス原則
- 育成必須化を避ける
- 進化・凸の達成感は強く、日課拘束は弱く
- 強さの差は小さく、演出の差は大きく

## 実装順 (推奨)
1. 詳細画面導線 (長押し)
2. 重複カードの保持・ロック
3. 凸 (`0-5`)
4. 変換 (変換先選択)
5. 進化演出
6. 強化簡易化
7. 同一カード複数編成の可否を設定化
